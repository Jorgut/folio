# Information Design · 信息设计

> Edward Tufte 等先驱的信息设计原则，直接应用于展示层排版。
> 核心问题：你的 deck 是清晰传达了信息，还是用装饰掩盖了信息不足？

---

## 1. 数据墨水比 (Data-Ink Ratio)

**定义：** `数据墨水比 = 用于展示数据的墨水 / 总墨水`

简单说：页面上的每一滴墨水都应该传递信息。如果去掉某个元素不影响理解——删掉它。

**为什么：** 装饰（多余的边框、背景色、渐变、图标）在消耗观众的注意力。每增加一个装饰元素，核心信息的接收率就下降一分。

**在 Folio 中的应用：**

```css
/* ✅ 高数据墨水比：只有必要的分隔线 */
.data-table th, .data-table td {
  border: none;
  border-bottom: 1px solid var(--hr);
  padding: 8px 16px;
}

/* ❌ 低数据墨水比：网格线 + 背景色 + 边框 = 3 倍无关视觉噪音 */
.data-table th, .data-table td {
  border: 1px solid #ddd;
  background: #f9f9f9;
  padding: 8px;
}
```

- Stats 布局不用装饰背景，数字本身是英雄
- 正文区域不用渐变/纹理
- Minimal 风格的哲学基础：去掉一切不必要的东西

---

## 2. 信噪比 (Signal-to-Noise)

**定义：** `信噪比 = 有效信息 / 无关信息`

和数据墨水比类似，但更广——"信号"包括信息本身，"噪声"包括所有不传递信息的元素（冗余标签、装饰性动效、不必要的颜色）。

**在 Folio 中的应用：**

| 场景 | 信号 | 噪声 |
|------|------|------|
| Stats 数字 | 数字本身 | 数字周围的装饰背景、图标 |
| 正文区域 | 文字内容 | 背景渐变、纹理、多余的引用线 |
| 导航点 | 当前在哪页 | 导航点闪烁/缩放动效 |

**决策规则：** 如果一个元素不能通过"它传达了什么信息"的测试——删掉。

---

## 3. F 型阅读模式

**发现：** 眼动追踪研究表明，用户浏览内容时呈 F 型模式：
1. 水平横读顶部（F 的第一横）
2. 向下扫左侧（F 的竖线）
3. 再水平横读中间区域（F 的第二横）

**为什么：** 这是互联网阅读行为的基本模式。你的 deck 的布局应当顺应这个模式，而不是对抗它。

**在 Folio 中的应用：**

- 最重要的信息在左上（F 的起点）
- 标题要能在第一眼抓住注意力
- 列表/要点用醒目的项目符号——帮助眼睛快速定位竖线位置

```html
<!-- ✅ F 型友好的 Stats 布局 -->
<section data-layout="stats">
  <div class="stat">
    <span class="stat-value">35%</span>   <!-- 左上：最重要的数字 -->
    <span class="stat-label">营收增长</span>
  </div>
  <div class="stat">
    <span class="stat-value">120 万</span> <!-- 右上 -->
    <span class="stat-label">新增用户</span>
  </div>
</section>
```

## 4. 倒金字塔结构

**原则：** 最重要的信息在最前面，然后是支持细节，最后才是背景信息。

**为什么：** 观众可能在你讲到一半时被打断。如果你的"重要结论"在最后一页——他们永远看不到。

**在 Folio 中的应用：**

```
封面：结论/主张（最重要的信息）
  → 副标题：关键数据
    → 正文：支持细节
      → 背景：为什么会这样
```

一封 deck 的封面必须给出"你要说什么"。不要在封面放公司 Logo + 演讲者姓名 + 日期——那是浪费第一页。

---

## 5. 排版信息层级

**原则：** 文字大小应区分出明确的信息层级，让读者一眼判断优先级。

**为什么：** 如果所有文字大小一样，用户无法判断什么是重要的。好的信息层级是"可扫描的"。

**在 Folio 中的实现：**

| 层级 | 视觉权重 | 语义 |
|------|---------|------|
| H1（封面） | 最高 | 品牌/主张/结论 |
| H2（章节） | 高 | 新概念/新章节 |
| H3（区块） | 中 | 子主题 |
| 正文 | 低 | 详细内容 |
| 元数据 | 最低 | 日期/作者/来源 |

```css
/* 字号差 ≥ 6:1 确保层级清晰 */
h1 { font-size: clamp(3rem, 6vw, 6rem); }   /* ~80px */
p  { font-size: clamp(0.9375rem, 1.1vw, 1.125rem); }  /* ~16px */
/* 比例 = 80/16 = 5:1，接近 6:1 目标 */
```

---

## 6. 图表完整性

**原则：** 数据展示的诚实性。不要用视觉技巧歪曲数据。

**在 Folio 中的应用：**

- 坐标轴从 0 开始（防止放大差异制造假趋势）
- 长宽比反映数据比例（不要通过拉伸图表来强调效果）
- 不使用 3D 图表（3D 歪曲了数值的对比——前面的看起来比后面的大，即使数值相同）
- 颜色只用于传递信息，不是装饰（色盲友好的调色板）

---

## 总结：信息设计检查清单

- [ ] 数据墨水比：去掉一个元素会损失信息吗？不会？删掉。
- [ ] 信噪比：每个元素都是"信号"吗？有"噪声"元素吗？
- [ ] F 型：最重要的信息在左上吗？
- [ ] 倒金字塔：封面的第一句话给结论了吗？
- [ ] 信息层级：只看字号能判断内容优先级吗？
- [ ] 图表诚实：坐标轴从 0 开始吗？没有用 3D 图表吗？
