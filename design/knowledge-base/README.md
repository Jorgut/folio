# Knowledge Base · 设计知识库

> 这里不是文档，是这个 skill 的设计基因。
> 
> **菜鸟**：不需要读这些，直接看 SKILL.md 4 步出 deck。
> **设计师**：理解这些原则 → 在 Folio 中做出更好的设计决策。

## 读什么

| 文件 | 解决什么问题 | 读完能做什么 |
|------|------------|------------|
| `gestalt.md` | 用户为什么把某些元素看成一组？ | 控制版面分组、层级、视觉流 |
| `ux-laws.md` | 交互决策有科学依据吗？ | 精通导航层级、触控目标、信息组块 |
| `information-design.md` | 怎么排版才能让信息有效传达？ | 控制数据墨水比、信噪比、阅读模式 |
| `accessibility.md` | 你的 deck 有没有排除某些用户？ | 达到 WCAG AA 标准，覆盖键盘/屏幕阅读器 |

## 这些原则在 Folio 里的位置

每条原则在 `engines/` 和 `design/style-guide.md` 中都有对应实现。原则解释"为什么"（why），引擎和风格参数解释"怎么做"（how）。
